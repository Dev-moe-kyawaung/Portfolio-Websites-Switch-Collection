FAQ

Frequently Asked Questions

What technologies do you specialise in?
My primary stack is Kotlin with Jetpack Compose for the UI layer, MVVM/Clean Architecture for structure, Hilt for dependency injection, Retrofit + OkHttp for networking, Room for local persistence, and Firebase for backend services. I also work with GitHub Actions for CI/CD and ML Kit for AI features.
How long does a typical Android project take?
A single-feature MVP takes 3–5 weeks. A production-ready multi-screen app with backend integration typically takes 8–12 weeks. Enterprise projects with team involvement are scoped individually. I provide a detailed timeline breakdown before any engagement begins.
Do you handle Play Store submission?
Yes. I handle the complete release workflow — keystore generation, ProGuard configuration, bundle signing, store listing assets, and initial submission. I also set up fastlane or GitHub Actions for automated future releases.
Can you work with existing codebases or teams?
Absolutely. I'm experienced in joining mid-project teams, performing code audits, migrating legacy Java codebases to Kotlin, and introducing modern architecture patterns incrementally without breaking existing functionality.
What is your availability and time zone?
I'm based in Tachileik, Myanmar (GMT+6:30). I'm flexible and regularly coordinate with teams in Europe, South Asia, and Southeast Asia. I aim to have a 4-hour overlap window with your core team hours.
Do you offer post-launch maintenance?
All packages include a free support window after launch. For ongoing maintenance — SDK updates, Android OS compatibility, bug fixes, and feature iterations — I offer monthly retainer agreements tailored to your team's needs.
